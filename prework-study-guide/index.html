HTML

  • The head element contains info about the webpage. Kind of like the brain being the control of the body
  • Within the head element lives the metadata. this is information aabout the page that browsers use. like the page language and viewport settings
  • The body element represents the visible content shown to the user. Similar to how you can visually see someone move their legs, but not see the internal order given to cause that movement.
  • Header's need to be in order, starting with h1 and ending with h6. There should only be one h1 and it should be the largest header.
  • you can make lists with ul, ol, and li. ul creates bullet point lists, ol creates numbered, or ordered lists, and li will be the actual listed items within the ul or ol

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch name: creates a new branch and switches to it.
  • git checkout branch name: switches to a branch. example: git checkout main
  • git add -A: stages changes made on a feature branch. This commits changes to the working branch.
  • git commit -m "added changes to HTML file": -m associates a message with our commit. Commit messages should be short and descriptive for changes implemented. ONce commited our git repository is now up to date with our working directory.
  • git pull: used to fetch and download content from a remote repository and immediately update the local repository to match that content.
  • git push: used to upload local repo content to a remote repo.
  • Repo: another name for a repository. Repositories are like safes for all of your coding projects.
